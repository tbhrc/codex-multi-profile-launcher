# Fresh Session Handoff

This document is the first-stop context packet for a future agent investigating or maintaining this repository.

## Current Purpose

This repo is a small macOS bridge for running two isolated Codex desktop profiles side by side.

It is not a task system, agent operating system, account switcher, credential manager, or OpenAI product. The task workspace or orchestrator remains the source of truth. This repo only provides:

- profile-specific `CODEX_HOME` directories
- profile-specific macOS launcher apps
- simple wrapper execution
- local execution evidence under `runtime/outputs/`
- documentation and scripts for backup, transfer, and maintenance

## Verified Local Facts

These are facts verified during the original build and later maintenance passes. Re-check them if they matter to a future change, because installed apps can move.

- Installed desktop app path used by launchers: `/Applications/ChatGPT.app`.
- App binary path used by wrappers: `/Applications/ChatGPT.app/Contents/MacOS/ChatGPT`.
- C1 launcher app: `~/Applications/Codex C1 Business.app`.
- C2 launcher app: `~/Applications/Codex C2 David.app`.
- C1 launch argument: `--user-data-dir=$HOME/Library/Application Support/Codex-C1-Business`.
- C2 launch argument: `--user-data-dir=$HOME/Library/Application Support/Codex-C2-David`.
- Regular app remains separate from both profiles.
- `CODEX_HOME` isolates CLI/config/runtime state, while `--user-data-dir` isolates desktop login and app state.

## Key Decisions

1. Keep the system boring.
   The repo is intentionally a bridge, not a full orchestration layer.

2. Use `C1` and `C2` as short codes.
   The earlier draft used longer or less useful codes. `C1` means Codex Business; `C2` means Codex David.

3. Do not duplicate the task system.
   Task intake, review, routing, tracker updates, and synthesis belong in the user task workspace or orchestrator.

4. Do not copy credentials.
   `auth.json`, OAuth stores, cookies, API keys, and desktop browser session state must never be copied, printed, or committed.

5. Desktop project containers are not reliably portable.
   Local sessions and continuity files can be backed up and merged, but visible desktop sidebar projects may need to be re-added manually.

6. Do not continuously copy live SQLite databases.
   The background watcher synchronizes safe session/history/memory artifacts only. Live SQLite copying can corrupt locks or lose concurrent updates.

7. Verify the actual desktop data root.
   A profile can have correct files on disk while the visible app is using another `--user-data-dir`. Check the running profile-specific argument before diagnosing missing projects.

6. Public README must be agent-friendly.
   The README includes a direct prompt a user can paste into a coding agent.

7. Public README must be legally clear.
   The README states this is unofficial, not affiliated with OpenAI, and may be changed or removed at OpenAI's request.

## Lessons Learned

- `CODEX_HOME` alone is not enough for simultaneous desktop sessions. The desktop app also needs a separate `--user-data-dir`.
- Separate launchers are better than manual logout/login flows because both profiles can stay active at the same time.
- The regular ChatGPT/Codex desktop app continues to use its default profile unless started through one of the wrapper apps.
- Custom icons matter operationally because Dock visibility is part of the workflow.
- Do not publish raw screenshots without checking for personal handles, account names, avatars, and usage data.
- Runtime files such as `runtime/active_worker.json` can change during normal use. Do not treat them as source changes unless the task is specifically about runtime state.
- Telemetry totals should come from Codex session metadata, not estimates from the visible conversation.
- The public repo should remain installable and forkable without exposing the user's private FolderDesk paths.

## Validation Commands

Use these before committing meaningful changes:

```bash
python3 tools/aosctl.py validate --verbose
python3 -m unittest tests/test_aosctl.py
bash -n scripts/*.sh wrappers/*.sh
```

For launcher-specific work:

```bash
bash scripts/install-macos-launchers.sh
open -n "$HOME/Applications/Codex C1 Business.app"
```

Verify only the narrow process path or user-data argument needed. Avoid broad process dumps that may expose unrelated secrets.

## Known Runtime Caveats

- `runtime/active_worker.json` records active worker state and may be modified by validation or activation workflows.
- `runtime/outputs/` is execution evidence, not the task source of truth.
- If a task belongs in the user's task workspace, document the boundary and stop instead of expanding this bridge.
- If a task asks for credentials, `auth.json`, cookie copying, account cycling, quota switching, or automatic credential movement, stop and report.
